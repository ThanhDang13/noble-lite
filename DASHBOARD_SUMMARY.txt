## Architecture Highlights

### ProcessRunner Pattern
All commands use threaded subprocess execution:
- stdout/stderr streamed via queue to main thread
- No GUI freezing during blocking operations
- Safe Tkinter updates (only main thread touches widgets)
- Process lifecycle management (start/stop/PID tracking)

### Privilege Escalation
Commands needing root (insmod, installer.sh, timectl.sh):
1. Try pkexec (native system dialog) first
2. Fallback to sudo -S with GUI password prompt
3. Password never logged or stored in memory

### Logging
All operations → logs/dashboard_YYYYMMDD.log:
- timestamp + level + module + message
- Color-coded in Logs panel (ERROR=red, WARNING=orange)
- One file per day, viewable from GUI

### Safety
- Destructive scripts (cleanup.sh, installer.sh) require confirmation
- Kernel operations warn about VM-only usage
- No script runs as root by default - per-command escalation only

## Usage Quick Reference

**Navigation:** Click sidebar items to switch panels

**Kernel Module:**
- Load → requires root, shows pkexec/sudo prompt
- Unload → requires root
- Status updates every 5 seconds in sidebar
- dmesg shows last 50 module-related messages
- Read /proc/process_monitor shows live process list from kernel

**Process Manager:**
- All operations are one-shot (run and exit)
- Kill process prompts for PID
- Output shows in scrollable console

**File/Network Monitors:**
- Watch modes are blocking (stop via "Stop Watch" button)
- File watch uses inotify (kernel events)
- Network watch uses configurable interval (1-60s)

**Socket Server:**
- Start on any port 1024-65535 (no root needed)
- Shows PID while running
- Real-time output shows client connections
- Stop terminates server gracefully

**Socket Client:**
- Connects to server, sends message, shows response
- One-shot operation (connects, sends, closes)

**Scripts:**
- ⚠ Destructive scripts (cleanup, installer) require confirmation
- 🔒 Root scripts (installer, timectl) show auth prompt
- All parameters filled via GUI forms (no CLI syntax needed)
- Output streams in real-time

**Logs:**
- Shows today's log file
- Filter by module name or any keyword
- Refresh button reloads from disk

## Demo Workflow

Once tkinter is installed:

1. Launch: `python3 launch_dashboard.py`
2. Click "Kernel Module" → Load (enter password if prompted)
3. Click "Read /proc/process_monitor" → see process list from kernel
4. Click "Process Manager" → List Processes → see userspace view
5. Click "Socket Server" → port 8080 → Start Server
6. Click "Socket Client" → host 127.0.0.1, port 8080 → Send Message
7. Click "Scripts" → select backup.sh → fill params → Run
8. Click "Logs" → see all operations logged
